## 项目简介
随着互联网、社交媒体、电商平台快速发展，网络每日产生海量非结构化中文文本，涵盖用户商品评论、社交动态、行业资讯、短视频留言等多种类型。传统单机自然语言处理程序依靠本地硬件完成计算，存在明显短板：本地硬盘存储容量有限，无法承载百万级大规模文本语料；无多副本备份机制，数据丢失风险高；串行计算执行速度慢，批量分词、统计任务耗时极长，难以支撑舆情监控、行业文本挖掘等工程场景。

针对单机NLP程序的局限性，本项目依托Hadoop、Spark分布式大数据计算集群，结合Python自然语言处理工具搭建完整离线文本分析系统。项目采用覆盖多领域真实中文评论数据集，完整打通分布式数据存储、原始文本清洗、中文精准分词、停用词过滤、词汇频次统计、可视化图表生成全链路流程。

系统开发与功能测试完成后，结合实验运行数据横向对比Hadoop、Spark、Flink三大主流大数据计算框架底层架构、运行模式、性能特点与适配业务场景，分析三类框架落地自然语言处理任务的优势与短板。依托理论知识与实操结果论证大数据分布式架构赋能NLP文本分析的可行性与实际价值，完整达成本次大数据编程课程设计全部理论研究与工程实践要求。

## 数据集详细说明
实验数据集文件名称为paper_text_data.csv，数据来源为真实网络用户评论文本，数据规范无大量脏数据，可完整模拟线上真实文本分析业务场景。
数据集基础参数：总数据量1201条中文评论文本，覆盖生活、职场、科技、娱乐、教育五大主流领域；
数据集字段释义：
text_id：单条文本唯一标识编号
comment：待分析核心中文评论文本
domain：文本所属业务领域分类标签
sentiment：文本情感正负标注标签

程序完整执行完毕后，所有输出结果统一存放至项目data目录，包含三类文件：词汇词频统计结果文本、TOP20高频词汇分布柱状图、全文本语义中文词云图，全部分析结果统一归档留存。

## 四大核心功能详细介绍
### 1 文本预处理清洗模块
程序读取CSV原始数据集后，第一步执行数据清洗过滤，自动剔除文件表头、空白行、字段缺失残缺无效数据；调用jieba中文分词工具对完整评论文本逐句拆分独立词汇；过滤无实际语义的单字、语气助词、连接虚词等干扰词汇，仅保留具备分析价值的实体词汇，大幅提升后续词频统计、可视化分析准确性。

### 2 全量词汇词频统计模块
文本分词、无效词汇过滤完成后，程序遍历全部有效词汇，统计每一个词汇在全部1201条文本中的出现总频次，按照出现次数从高至低降序排序，最终将完整统计结果写入本地txt文本，生成标准化可查阅词频报表。

### 3 数据可视化绘图模块
基于词频统计结果完成两类可视化图表绘制：选取出现频次排名前20的高频词汇生成柱状图，直观对比不同词汇出现频次差距；生成全文本中文词云图，词汇出现次数越高，渲染字体尺寸越大，形象展示整体文本语义分布特征，图表分辨率统一设置300DPI，满足课程设计论文配图清晰度标准。

### 4 分层模块化工程架构
项目采用模块化开发架构，全局配置、原始数据、文本计算、可视化绘图、自动化运行脚本完全解耦分离，模块间无强耦合关联。后期如需修改文件存储路径、调整分词过滤规则、新增文本分析算法，仅需修改对应独立模块代码，不会干扰其余功能正常运行，项目拓展性、可维护性优秀。

## 三大主流大数据框架NLP场景适配分析
结合本次项目实操运行数据与大数据、自然语言处理理论知识，对Hadoop、Spark、Flink三款主流分布式计算框架适配文本挖掘、NLP任务的特性与适用场景总结如下：
1. Hadoop
依托HDFS高可靠分布式存储与MapReduce离线批处理计算模型，系统运行稳定性强、数据多副本容错机制完善，集群部署、日常运维操作简单，更适合海量历史归档语料、大规模静态文本离线批量分析，是传统离线文本挖掘项目主流选型。

2. Spark   2. 火花
基于内存迭代式计算架构，减少磁盘反复读写IO开销，多轮循环计算性能远优于Hadoop MapReduce，迭代运算速度提升明显，适用于精细化文本特征筛选、交互式深度文本挖掘、准实时小批量NLP分析场景。

3. Flink
核心优势为低延迟流式实时计算，支持7×24小时不间断持续数据流接收与处理，专门适配实时评论抓取、线上动态舆情监控、实时内容识别等对数据处理时效性要求极高的自然语言处理业务。

三大框架技术定位、性能优势互补，实际工程可根据文本数据体量、业务实时性需求组合搭配使用。

## 常见故障与完整解决方案
### 故障1 HDFS无法创建目录、上传写入数据集
故障诱因：集群启动后默认开启安全模式，安全模式下限制全部文件写入操作
```bash   ”“bash
hdfs dfsadmin -safemode leaveHDFS dfsadmin -safemode leave
```

### 故障2 运行Python程序提示第三方模块缺失
故障诱因：依赖库未完整安装、安装过程中断缺失部分包
```bash   ”“bash
cd ~/BigData-NLP-Code
pip3 install -r requirements.txtPip3 install -r requirements.txt
```

### 故障3 柱状图、词云图中文渲染为方框乱码
故障诱因：Ubuntu系统未安装中文字体，或config.py配置文件内字体路径参数与本机真实字体路径不匹配
解决方式：安装文泉驿微米黑中文字体，打开config.py修改CHINESE_FONT变量，替换为本地真实字体存放路径。

### 故障4 程序读取/写入文件提示路径不存在
故障诱因：代码内相对路径书写错误、HDFS分布式目录未提前创建
解决方式：逐行核对config.py内部全部本地文件路径、HDFS分布式路径，路径有误则修正；缺失目录执行hdfs mkdir命令新建。

### 故障5 Spark与Python版本不兼容，启动交互报错
故障诱因：Spark 2.4.8旧版本对高版本Python兼容性较差，直接混用会触发启动异常
解决方式：搭建Python虚拟环境隔离版本，不修改系统全局默认Python环境。
